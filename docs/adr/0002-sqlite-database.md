# ADR-0002: 使用 SQLite 作为主数据库

## 状态
已接受

## 背景
需要选择项目的主数据库方案。

## 决定
使用 SQLite（better-sqlite3），单文件数据库（data.db）。

## 原因
- 项目初期数据量小，SQLite 完全够用
- 零配置，部署简单，不需要额外的数据库服务
- better-sqlite3 是同步 API，Express 代码更直观
- 开启了 WAL 模式提升并发读性能

## 后果
- 不适合高并发写入场景（但当前规模足够）
- 数据库文件和应用在同一台机器，备份需要复制文件
- 未来如果需要扩展，可以迁移到 PostgreSQL，但需要改写所有 SQL
