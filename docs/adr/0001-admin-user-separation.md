# ADR-0001: 管理员和用户使用独立表和独立 JWT 密钥

## 状态
已接受

## 背景
系统有两类身份：前台用户（users）和后台管理员（admins）。需要决定是共用一张表还是分开。

## 决定
使用**独立的表**（users / admins）和**独立的 JWT 密钥**（JWT_SECRET / ADMIN_JWT_SECRET）。

## 原因
- 用户和管理员的数据结构差异大（用户有手机、VIP；管理员有角色、权限）
- 安全隔离：管理员 token 泄露不会影响用户，反之亦然
- 管理员数量极少，不需要复杂的用户系统功能
- 登录入口完全分离（/login vs /admin-login）

## 后果
- 两套认证中间件（verifyToken / verifyAdminToken）
- 不能简单地把用户"升级"为管理员，需要单独创建
- 前端路由保护也需要分别处理
